# 做研究 - 第一部分：系统安全生存简明指南

<!-- more -->

系统安全是通过安全保障技术来防御计算机系统免受漏洞和攻击等威胁的实践。这听起来很崇高——确实如此——但让我们坦诚一点：系统安全出名地难以入门、进展艰辛，而且通常过于脆弱而难以生存。

那么我们为什么还要做它呢？

我相信这是因为我们真正被它吸引。但说真的——当这么多力量似乎都在反对它时，我们为什么还在做系统安全？

## 严酷的现实

事实是，系统安全面临着严峻的挑战：

- 风险投资很少青睐我们。
- AI 科学部门并不真正需要我们。
- 我们的研究从根本上来说既艰难又缓慢。

每一天，我都不得不说服自己留在这个领域——明知它不太可能带来领先公司的高薪，或者成为顶尖大学教职的捷径。

然而，我们仍在这里。

## 重建信心

正如肯尼迪总统曾经问道："为什么选择这个作为我们的目标？他们可能会问，为什么要攀登最高的山峰？为什么在35年前飞越大西洋？为什么莱斯大学要对阵德克萨斯大学？"

我们选择系统安全不是因为它容易，而恰恰是因为它艰难。因为当下一波新技术浪潮崩溃时——当它们关于"信任"的承诺破灭时——系统安全依然会在那里。

## 讲故事、新自由主义和生存

我们还必须认识到我们所生活的世界。在新自由主义下，大学、教职人员和学生都被资本驱动。网络安全也必须——至少部分地——与资本的利益保持一致，如果它想要生存的话。事实上，网络安全一直通过保护底层系统来帮助确保资本的盈利能力。

特别是系统安全，与这些生态系统深度绑定。我们的使命是构建跨越整个技术栈的信任链——从底层软件到 AI 应用。在其核心，系统安全由几个基本技术驱动（待续）：

- 隔离（Isolation）——分离关键组件
- 证明（Attestation）——证明系统的完整性

系统安全不是关于纯密码学、为了数学证明而进行的数学证明，或者最新的 AI/ML 突破。它关乎信任——就像把资本放入保险箱，通过不可破坏的信任链来保护价值。

## 讲故事、国家安全和生存

另一种讲故事的形式是成为国家安全本身的一部分。网络空间现在被认为是冲突的"第五领域"，与陆地、海洋、空中和太空并列。网络空间的国家安全政策是政治意愿的自然延伸。我相信在保持防御姿态的同时维持先进攻击能力的原则。从这个意义上说，系统安全研究必须同时考虑进攻和防御。我们的防御必须强大而有韧性，但我们对攻击的理解必须更加先进——以保持战备状态、可信度和能力。

## 创新性和向前发展

系统安全的生存超越了资本和国家安全的关切，但它通过持续的技术创新而进步。以下是正在塑造安全计算未来的五大研究问题：

- 状态性（Statefulness）
- 大规模系统理解
- AI 基础设施安全
- 全链漏洞利用
- 硬件无关的软件/通信协议

系统安全必须发展以应对资本和国家安全日益增长的关切，因为技术越来越成为经济和地缘政治稳定不可或缺的一部分。同时，系统安全的有意义进展取决于解决基本问题——如状态性、大规模系统理解和全链漏洞利用。通过解决这些核心挑战，系统安全可以变得更加适应性强和有韧性，更好地装备以在日益竞争的环境中茁壮成长。系统安全万岁！
