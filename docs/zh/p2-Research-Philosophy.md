# 做研究 - 第二部分：如何做系统安全研究？

## 什么是系统安全？

我们的目标是攻破和保护那些利用硬件和软件提供广泛使用服务的资产，例如浏览器、编译器/解释器、网络协议、操作系统、虚拟机监视器、可信执行环境和处理器，这些都基于定义谁将被信任或不被信任的威胁模型。


### 算法

- 纯密码学是数学。应用密码学与系统和软件安全有重叠——我们应用非常基础的密码学作为提供机密性的机制。

- 隐私保护计算关注数据计算、交换和存储过程中的隐私问题。

- 可信 AI 从根本上旨在攻破和保护模型的高维函数。

### 软件和硬件

- 系统人员构建系统并将安全视为他们系统应该具有的属性之一。相反，安全人员不太关心业务。

- 软件工程研究围绕软件开发生命周期（SDLC）。类似地，他们也将安全视为他们希望在 SDLC 中拥有的属性之一。安全人员强调漏洞和利用。

### 细分系统

- 云、嵌入式设备和个人计算机完全连接，这需要安全的协作计算。

- 区块链是关于共识的。在实践中，区块链在金融领域有丰富的应用，这令人印象深刻。

- AI 系统与传统 IT 系统不同，这是由于整个软件和硬件栈的变化。

## 想法从哪里来？

一个想法由一个问题和一个解决方案组成。

- 业务指标驱动：想法来自工业界。我们在工业界工作，或者我们从工业界获得礼物资金。

- 国家研究资金驱动：想法由国家资助机构塑造，他们负责实施国家战略。

- 论文驱动：我们可以在阅读论文后阅读并提出新想法。然而，会有延迟。一方面，研究论文的发表比作者提出想法晚两年。另一方面，即使一些程序委员会成员可以检查提交给会议的所有有趣论文，也有大约半年或一年的延迟。

- 研究活动驱动：第一个想法可能来自我们的导师。后来，实际的想法来自我们的研究活动，我们应该成为批判性思考者。

查看[这篇文章](https://medium.com/digital-diplomacy/how-to-look-for-ideas-in-computer-science-research-7a3fa6f4696f)来学习研究的模式。

## 有太多想法，如何决定？

安全研究社区已经发展得很大。由于廉价的资源（计算能力、ChatGPT 等），更多的博士生产生了更多的论文。内卷正在进行，我们必须通过建立我们的品牌来快速提升我们的知名度。在实践中，我们必须选择一个我们可以持续研究四到五年的主题。询问我们的导师。

就具体主题而言，我们必须评估我们的想法，因为并非所有想法都值得我们付出努力。

原则 1：选择一个好问题

- 先有问题，然后才有解决方案
- 新问题总是更好
- 想法易于理解且有明确的好处
- 如果创新性很小，应该有很大的安全影响

原则 2：选择一个难题

- 解决一个简单的问题不需要论文
- 采取合理的假设并提出几个解决方案
- 了解最优解：考虑计算理论并估计我们能做到的最好结果，然后做一个近似

原则 3：必须阅读相关工作以了解他们已经走了多远。请注意有多少研究论文和行业博客存在。你想引领研究（只有0-2篇论文）而不是跟随别人。不要傲慢，记住你的个人经验是有偏见的。

## 做研究的技巧

### 管理我们的时间

- 提前两周标记截止日期
- 多花30秒来避免30分钟的灾难
- 不要做很多上下文切换，这会浪费很多时间
- 在非第一作者项目上花费少于一天的时间
- 该忙的时候忙，该玩的时候玩
- 提交后休假和周末（对于博士生）

### 我们要做的就是我们要写的

- 审查论文，运行他们的系统，并总结（相关工作）
- 识别挑战并在理论上评估解决方案（挑战、见解、解决方案）
- 设计系统并在理论上评估设计选择（设计）
- [编码](p5-Summary-Research-SE.md)并编写评估脚本（实现、评估）
- 同时撰写论文并从他人那里获得反馈

### 做评估

- 全部在 Docker 容器中
- 使其并行化
- 首先获得快速结果，中长期结果用于写作，然后是提交前的长时间评估
- 首先追求多样性，然后是数量
- 如果我们确定，不要删除任何数据。只需购买更多存储！
- 不要让硬件休息。

## 写一篇好论文

- 讲一个结构良好的故事（金字塔原理）
- 进行以读者为中心的写作：直截了当且清晰
- 最小化缺点
- 写作过程
    + 起草：每个人都很棒！
    + 修订：焦点、支持、连贯性
        - 一个段落应该有一个主题句
        - 支持材料必须具体
        - 多个主题句应该有逻辑性
    + 编辑：语法、词汇选择、句子技巧
        + 注意 a/an/the、单数/复数、could/should
        + 为所有级别的审稿人选择词汇
        + 选择只有一个含义的词
        + 少用隐喻
        + 不要进行形式化分析
    + 校对：从"审稿人"那里获得第一轮反馈

### 以读者为中心写作的必要性

- 提交的论文太多但审稿人不够
- 截止日期太多，以至于审稿人最多只有4小时来弄清楚我们做了什么。节省他们的时间！
