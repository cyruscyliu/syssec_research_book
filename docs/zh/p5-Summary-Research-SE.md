# 做研究 - 第五部分：系统安全研究的工程实现

在有限的时间内扎实的实现使我们在研究中更具竞争力。


- 引入工件评估是为了克服"可重现性问题"。
- 安全社区是五年前的三倍。

系统安全研究需要扎实的实现。为了通过工件评估，我们最好花更多时间来提高可用性。快速而粗糙的实现节省了时间，但可用性较低，因此时间根本没有节省。为什么不在开发之初就考虑可用性呢？我们也被鼓励开源我们的工具，这需要更高质量的实现，工具中的缺陷数量更少。

系统安全研究需要时间。我们需要了解不熟悉的底层软件或硬件如何工作，尝试它，重新理解，重新尝试，等等。在我们一半的时间里，我们在编译东西，在剩下的时间里，我们在犯错误。

然而，随着社区的大幅增长，我们希望尽快实现我们的想法。我提出了一个两步模型。首先，我们构建我们的工具箱，并以快速而粗糙的方式实现我们的想法。这一步是为了让高年级学生或初级教授确保他们的想法可行。其次，我们遵循足够且快速的编码理念，也就是说，快速开发具有足够功能的工具。通过这种方式，我们可以通过工件评估而不过度工程化，直接开源我们的工具，并节省时间。

## 足够的功能

考虑最少的功能集，不仅包括核心功能，还包括使该工具易于使用、调试和评估的功能。

## 快速开发

软件工程有许多节省时间的最佳实践。

### 先设计架构再选择框架

设计和实现是不同的。通常设计展示了如何解决问题，我们需要选择最合适的方式来实现我们的设计。

### 控制时间和风险

避免过度工程化。请注意，您/其他人稍后将维护您的项目。使用简单的技术，不要炫耀。设计模式是我们可以尝试的，但要解决真正存在的一些问题。对于任何其他不确定的事情，制定备用计划。
