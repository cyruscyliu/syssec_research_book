# 做研究 - 第七部分：技术转化与落地实践


技术转移是我们研究的一部分，并将为我们的研究提供反馈。理论分析将在此省略。在实践中，（但我仍想将范围限制在系统和软件安全）我们研究的真正影响是我们的论文被接受和我们的提案获得资助的强有力标准之一。同时，任何技术转移的方式（技术转移并不意味着我们必须拥有一家初创公司）都会向我们展示我们的研究是否足够好以解决实际问题。除了一些实践经验外，由于许多原因，管理部门和企业也希望技术转移。我认为将技术转移作为我们研究的一部分是一个好主意，但我们必须小心。

## OpenToy 及其商业化

假设我已经开源了一个名为 OpenToy 的工具来支持一些安全分析。当我写这篇文章时，OpenToy 还不太成熟。我想把它作为一个例子，看看我们如何将其商业化。在下文中，我将列出商业化的方式并进行比较。希望我们能看到一个越来越成熟的 OpenToy 被转移。

### 谁使用，谁付费？

作为第一步，考虑你的用户和客户。

用户有时间但没有钱支付，而客户没有时间但有钱支付。开源工具有很多用户和很多客户并不总是正确的。有时，两者之一。通常，该工具的用户和客户非常有限。

OpenToy 的潜在用户：你自己、其他研究人员、自由职业黑客。他们对 OpenToy 提供的功能感兴趣。他们不会付费，但会接受引用你的工具。

潜在客户：想赚钱的公司里的安全分析师。将这些安全分析师与其他用户区分开来并不容易。我猜他们也不会付费，除非你有证据表明他们用 OpenToy 赚了很多钱（但我不认为赚很多钱是可能的），他们甚至不会让你知道他们使用了你的工具一段时间，否则他们就必须付费。我应该对他们更好地看待吗？不。我应该学会推动他们付费。

### 如何赚钱？

如果有很多用户和客户，你可以在软件中投放广告，要求他们捐赠，在一个月试用后出售许可证以启用所有功能，就如何使用该工具提供建议，并为付费客户提供高级服务。

如果没有，事情就变得简单了。你必须使用你的工具做一些事情，宣传该工具以吸引更多用户和客户。做漏洞赏金可能是一个好的开始。
